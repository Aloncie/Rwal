# Security Policy

## 🛡️ Supported Versions

Rwal is actively developed, and the latest version receives all security updates.  
Older versions may not be maintained, so please upgrade to the latest release.

| Version | Supported          |
| ------- | ------------------ |
| v0.1.0-alpha (latest) | :white_check_mark: |
| < v0.1.0 | :x: |

## 🐛 Reporting a Vulnerability

I take security seriously. If you discover a security issue in Rwal, please report it privately.

### How to Report

Use GitHub's **private vulnerability reporting** tool:
   - Go to the repository's **Issues** tab
   - Click **"Report a security vulnerability"**
   - Follow the instructions

**Do NOT open a public GitHub issue** – this could put users at risk.

### What to Include

- A clear description of the issue
- Steps to reproduce (if possible)
- Potential impact
- Any suggested fixes (optional)

### 📢 Disclosure Policy

- I'll acknowledge your report within 48 hours
- If the report is valid, I'll work on a fix
- You'll be credited in the release notes (unless you prefer anonymity)

## 🌐Current Security Focus

Rwal is in **active development**, and I'm currently focusing on:

- 🔄 Refactoring the menu system for better stability
- 🧪 Adding unit tests to prevent regressions
- 🌍 Cross-platform hardening
- 📦 Safe handling of external data (Wallhaven API)

Your help in making Rwal more secure is greatly appreciated. Thank you! 🙏
