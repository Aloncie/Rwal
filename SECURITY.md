# Security Policy

## 🛡️ Supported Versions

Rwal follows a **latest‑release only** support policy. This means:

- Only the **most recent release** receives security updates.
- If you are using an older version, please **upgrade to the latest release** to ensure you have all security fixes.

We do **not** backport security patches to previous releases. This keeps development focused and ensures that all users benefit from the latest improvements.

| Version         | Supported          |
|-----------------|--------------------|
| latest release  | :white_check_mark: |
| any older release | :x:              |

> 💡 Always check the [Releases](https://github.com/Aloncie/Rwal/releases) page for the newest version.

## 🐛 Reporting a Vulnerability

I take security seriously. If you discover a security issue in Rwal, please report it privately.

### How to Report

1. Use GitHub's **private vulnerability reporting** tool:
   - Go to the repository's **Issues** tab
   - Click **"Report a security vulnerability"**
   - Follow the instructions

**Do NOT open a public GitHub issue** – this could put users at risk.

2. Write to Aloncie@proton.me
   
### What to Include

- A clear description of the issue
- Steps to reproduce (if possible)
- Potential impact
- Any suggested fixes (optional)

### 📢 Disclosure Policy

- I'll acknowledge your report within 48 hours
- If the report is valid, I'll work on a fix
- You'll be credited in the release notes (unless you prefer anonymity)

## 🌐Current Security Focus

Rwal is in **active development**, and I'm currently focusing on:

- 🧪 Adding unit tests to prevent regressions
- 🌍 Cross-platform hardening
- 📦 Safe handling of external data (Wallhaven API)

Your help in making Rwal more secure is greatly appreciated. Thank you! 🙏
